    介绍与准备工作

    NineSenz 7/1/2023 NodejsNestjs

    # 1.1 什么是 NestJS

    NestJS 是一个用于构建高效且可扩展的后端应用程序的渐进式 Node.js 框架。它采用了现代的 TypeScript 编程语言，基于 Express.js 构建，并借鉴了 Angular 的设计思想，使得 NestJS 具有强大的依赖注入、模块化和面向切面编程等特性。这些特性使得 NestJS 在开发企业级应用、RESTful API、微服务和实时应用等场景下表现出色。

    # 1.2 NestJS 的特点和优势

    NestJS 具有许多特点和优势，使其成为许多开发者喜爱的框架之一。

    # 1.2.1 渐进式

    NestJS 允许你根据项目的需求逐步采用其特性。你可以从一个简单的 Express 应用开始，并逐渐引入更多的功能和模块，使其适应不同规模的项目。

    # 1.2.2 TypeScript 支持

    TypeScript 是 JavaScript 的超集，为 JavaScript 添加了静态类型检查。NestJS 完全支持 TypeScript，这使得代码更具可读性、可维护性，并且能在开发过程中发现许多常见的错误。

    # 1.2.3 依赖注入（Dependency Injection）

    NestJS 的核心概念之一就是依赖注入，这使得组件之间的解耦更容易，代码更易于测试和维护。通过使用依赖注入，我们可以在组件之间传递依赖项，而不是在组件内部直接实例化它们。

    # 1.2.4 模块化（Modularity）

    NestJS 使用模块来组织应用程序。模块是一个具有相关功能的逻辑块，它包含了控制器、服务、提供者等组件。模块使得应用程序的结构清晰，并且易于扩展和维护。

    # 1.2.5 强大的抽象层

    NestJS 提供了一套强大的抽象层，使得开发者可以更轻松地使用常见的 Node.js 模块和功能。它封装了 HTTP 请求处理、WebSocket、数据库连接等细节，使得开发者可以专注于业务逻辑而不是底层实现。

    # 1.2.6 全面的文档和活跃的社区

    NestJS 提供了详细的官方文档，包含了大量示例和教程。同时，NestJS 社区非常活跃，你可以在社区中获取帮助、分享经验，还可以找到许多开源的插件和库。

    # 1.3 安装 Node.js 和 npm

    要开始使用 NestJS，你首先需要安装 Node.js 和 npm。Node.js 是一个基于 Chrome V8 引擎的 JavaScript 运行时，而 npm 是 Node.js 的包管理器，用于安装、管理和发布 Node.js 模块。

    你可以在 Node.js 官方网站（https://nodejs.org/）下载适合你操作系统的安装包，并按照提示进行安装。

    # 1.4 创建新项目

    安装 Node.js 和 npm 后，你可以通过以下命令来检查是否成功安装：

    node -v
npm -v

    接下来，我们创建一个新的 NestJS 项目。首先，在你想要存放项目的文件夹内，打开终端（或命令提示符）并执行以下命令：

    mkdir my-nest-project
cd my-nest-project

    2

    # 1.5 初始化 NestJS 项目

    在项目文件夹中，使用 NestJS CLI 初始化一个新的 NestJS 应用。执行以下命令：

    npx @nestjs/cli new backend
cd backend
    1
    2

    npx 是 npm 5.2.0 以后版本附带的工具，它能够帮助你在不全局安装模块的情况下运行 Node.js 包。@nestjs/cli 是 NestJS 的官方 CLI 工具，用于创建、构建和管理 NestJS 项目。

    执行上述命令后，NestJS CLI 将会引导你完成一系列配置选项，如选择包管理工具（npm 或 yarn）、选择要启用的功能等。根据你的实际需求进行选择。

    完成初始化后，你的 NestJS 项目就已经创建成功了，你可以进入项目文件夹并开始编写代码。

    # 1.6 项目目录结构解释

    在初始化项目后，你会看到 NestJS 为你创建了一系列文件和文件夹。下面简要解释一下这些文件和文件夹的用途：

    • src：该文件夹是你编写应用程序代码的主要目录。
      • src/main.ts：这是应用程序的入口文件，它会启动 NestJS 应用。
      • src/app.module.ts：这是应用程序的主模块，它是所有其他模块的根模块。
      • src/app.controller.ts：这是一个示例控制器文件，用于处理 HTTP 请求。
      • src/app.service.ts：这是一个示例服务文件，用于提供数据和业务逻辑。
      • test：该文件夹包含了一些测试文件，用于执行单元测试和端对端测试。

    在后续的章节中，我们将更详细地学习这些文件和文件夹，并开始编写我们的第一个 NestJS 应用程序。

    以上是第一章的内容，我们介绍了 NestJS 的概念、特点和优势，以及如何准备开发工作
