    Nest核心组件 - 提供者 Providers

    NineSenz 7/3/2023 NodejsNestjs

    # 2.2 提供者（Providers）

    提供者在 NestJS 中用于处理控制器和其他服务的依赖项。它们是可注入的类，可以在其他类中使用依赖注入的方式引用它们。

    在 NestJS 中，提供者的常见例子包括服务、数据库连接、第三方 API 客户端等。

    # 2.2.1 创建一个提供者

    让我们来创建一个简单的服务作为提供者。首先，在 src 目录下创建一个新的文件 cats.service.ts，并编写以下代码：

    import { Injectable } from '@nestjs/common';

@Injectable()
export class CatsService {
  private readonly cats: string[] = [];

  create(cat: string) {
    this.cats.push(cat);
  }

  findAll(): string[] {
    return this.cats;
  }
}

    在上面的代码中，我们使用了 NestJS 提供的 Injectable 装饰器来定义一个提供者类。@Injectable() 表明该类是一个可注入的类。

    然后，我们在 CatsService 中定义了两个方法：create 和 findAll。create 方法用于添加新的猫咪数据到 cats 数组中，而 findAll 方法返回 cats 数组中的所有猫咪数据。

    # 2.2.2 在控制器中使用提供者

    现在我们已经创建了一个简单的提供者 CatsService，接下来我们将在控制器中使用它。

    在之前创建的 cats.controller.ts 文件中，我们需要引入 CatsService，并在控制器的构造函数中通过依赖注入的方式使用它。修改 cats.controller.ts 文件，添加以下代码：

    import { Controller, Get } from '@nestjs/common';
import { CatsService } from './cats.service';

@Controller('cats')
export class CatsController {
  constructor(private readonly catsService: CatsService) {}

  @Get()
  findAll(): string[] {
    return this.catsService.findAll();
  }
}

    在上面的代码中，我们通过构造函数中的参数 private readonly catsService: CatsService 将 CatsService 注入到了 CatsController 中。这样，我们就可以在 CatsController 中使用 catsService 来调用 CatsService 提供的方法。

    当我们发送 GET 请求到 /cats 路径时，CatsController 的 findAll 方法会调用 CatsService 的 findAll 方法，并返回所有的猫咪数据。
